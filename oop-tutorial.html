Object Oriented Programming (OOP)

Objects

Object oriented programming is one of the fundemental precepts of Java, and understanding it is integral to using the language effectively.

OOP is a type of programming that revolves around objects. Objects in Java are a lot like real life objects, with attributes (variables) and functions (methods).

Each object, or instance, is created from a class, which acts as a blueprint for the object, providing it with variables and methods to use. However, the individual variable values between different objects of can vary greatly, as each instance has its own set of values.

For example, a football and a basketball would both be objects of the Ball class.
Another example, Bob and George would both be objects of the Human class

OOP is very useful for the programming principle of Don't Repeat Yourself, or DRY, as you can use one Class for many different purposes.

Access

All datatypes, primitive or Classes, and methods in Java all have an access level, which defines which other classes can access the thing directly.

Class access means access within the same class.
Package access means access within the same package, or defined group of classes.
Subclass access means any subclass of the original class.
World access means any Class.

Parameters

A method can have parameters, which are variables used in the method that are given values upon method call. Parameters act as a sort of input for a method. These are very useful for flexible methods that return different things depending on their input.

Return Type

Each method must have a return type, which is its variable output. The return type can be void, where it returns nothing. The return type can also be a primitive type, like int, or an object reference. Each method may only return one thing, though.

To use a return method, simply call the method wherever its return value is used and Java will run the method and use the value returned in the place of the method call.

Overloading

Another thing to note about methods is that they can be overloaded, which means an additional method declaration exists with the same name but different parameters. These two methods are called the same thing, but can do different functions.

For example, an area method can be overloaded to take one integer or two integers, with the single integer input method returning the area of a square with given side, and the two integer method input returning the area of a rectangle with given length and width.
Void methods can also have overloads.

Constructor

One method that every class must have in Java is a constructor, which runs upon object creation and does any variable initializing or other things that the object requires to function. You will have to implement these things yourself. A constructor is unlike any other method, as it is always declared as public, and does not have a return type. A constructor may also have overloaded methods.
Sometimes, you want to have basic constructor functions within an overloaded one. In this case, simply call the constructor with "[ClassName](parameter)" at the start of the other constructor.
One thing to note is that the program will still compile without any defined constructor. The program assumes that there exists a no-argument constructor. However, direct calls to the constructor or any overloads will not work.

Class Variables and Methods

Each method or variable can either belong to the class or the individual object. We call these class or instance methods/variables. Instance variables are created upon object creation, and each object has a copy. Class variables, on the other hand, belong to the class and are universal to all instances of the class. Class methods are similar, belonging to the class and not the instance object, but they simply cannot use instance variables.

Class variables and methods are declared in using the "static" keyword, which comes after the access keyword.
